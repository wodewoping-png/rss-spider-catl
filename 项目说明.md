# RSS Spider CATL — 学术文献自动采集与分析系统

## 项目概述

本项目是一套面向新能源/电池/碳中和领域的**自动化学术文献日报与周报生成系统**。每天定时从数十个顶级学术期刊的 RSS 源抓取最新论文，自动获取摘要，按主题分类，翻译成中文，最终生成可直接使用的 Word/Excel 周报。

---

## 整体流程

```
RSS 源 (feeds1211.txt)
        ↓
[1] 每日爬取 (spider0301.py)
    抓取昨天 + 前天的论文条目
    → output/news_with_abstract_YYYY-MM-DD.csv
        ↓
[2] 周报聚合 (weekly_aggrerate_with_abs.py)
    合并滚动 7 天的日报 CSV
    → output/weekly/weekly_news_with_abstract_YYYY-MM-DD.csv
        ↓
[3] 分类 + 翻译 (classify_weekly_onefile.py)
    GPT 翻译标题/摘要 → 中文
    语义嵌入 + GPT 按 classification.txt 分类
    → output/weekly/*_translated.xlsx / .docx
```

---

## 模块详解

### 1. RSS 爬取 (`spider0301.py` / `spider_abstract_1216.py`)

**目标：** 每天抓取学术期刊 RSS，只保留"昨天+前天"（UTC）发布的论文，并尽可能补全摘要。

**RSS 源覆盖（`feeds1211.txt`）：**
- Nature 系列（Nature、Nature Energy、Nature Materials、Nature Chemistry 等）
- ACS（Chemical Reviews、ACS Energy Letters 等）
- Science / Science Advances
- Wiley（Angewandte Chemie 等）
- Cell Press（Joule、Chem、Matter 等）
- ScienceDirect（Applied Energy、Energy Policy）
- RSC（Energy & Environmental Science 等）
- PNAS、National Science Review 等

**摘要获取策略（多层 fallback）：**

| 优先级 | 来源 | 适用出版商 |
|--------|------|------------|
| 1 | RSS description 直接解析 | Cell Press, OUP/NSR |
| 2 | Playwright 无头浏览器抓取页面 | Nature, RSC |
| 3 | Crossref API | 通用 |
| 4 | Semantic Scholar API | 通用 |
| 5 | OpenAlex API（含倒排索引重建摘要）| 通用 |
| 6 | PubMed eUtils API | 通用 |

**特殊处理逻辑：**
- **Wiley 日期校验：** Wiley RSS 日期常不准，用 Crossref/PubMed 的 canonical 发表日期进行二次校验。若 canonical 日期在最近 5 天内则保留，否则丢弃。
- **ScienceDirect 特殊源：** Applied Energy / Energy Policy 的 RSS 日期完全不可信，改为直接抓最新 N 篇，发布日期统一写为抓取前一天。
- **昨日缓存复用：** 读取前一天的 CSV，已有摘要的直接复用，避免重复调用 API。
- **重试机制：** 昨日 3 天内无摘要的条目，今日再次尝试补全。

**输出字段（CSV）：**
`title`, `link`, `source`, `published_str`, `pub_date`, `doi`, `last_author`, `abstract`, `abstract_source`

---

### 2. 关键词/语义过滤 (`best_filter_Qiaochu_with_abstract.py`)

**两层过滤：**
1. **规则匹配**：检测标题+摘要中是否含有指定关键词（CCUS、carbon capture、CO2、decarbonization 等）
2. **语义匹配**：对未命中的条目，用 `sentence-transformers/all-MiniLM-L6-v2` 计算语义相似度（阈值 0.55），过滤出相关文献

---

### 3. 周报聚合 (`weekly_aggrerate_with_abs.py`)

- 读取最近 7 天的日报 CSV，合并去重（以 DOI 或归一化 URL 为 key）
- 若同一篇文章在多天出现，优先保留信息更完整的版本（摘要更长者优先）
- 输出标准化周报 CSV 到 `output/weekly/`

---

### 4. 分类 + 翻译 (`classify_weekly_onefile.py`)

**分类体系（`classification.txt`，共 12 类）：**

| 类别 | 内容 |
|------|------|
| 光伏 | 太阳能电池与组件（钙钛矿、晶硅、有机光伏等） |
| 正极 | 可充电电池正极材料 |
| 负极 | 可充电电池负极材料（石墨、硅、硬碳等） |
| 电解质 | 固态/液态/凝胶电解质体系 |
| 非活性材料等 | 隔膜、集流体、热管理、BMS、电池回收等 |
| 其他储能器件 | 液流电池、超级电容器、金属空气电池等 |
| 热能 | 储热、热泵、热电材料等 |
| 核能 | 裂变/聚变反应堆、核燃料 |
| 氢氨醇 | 电解水制氢、绿氨、CO2 制甲醇等 |
| 产业降碳 | 钢铁/水泥/交通等行业低碳改造 |
| CCUS | 碳捕集、利用与封存 |
| 海底采矿 | 深海矿产开采 |
| 塑料回收 | 化学/机械回收、可降解塑料 |

**分类算法（三层混合）：**
1. **排除规则**：`classification.txt` 中"不包括"部分提取排除关键词，命中则阻止该类别
2. **包含规则**：命中 ≥ 2 个包含关键词则直接自动归类
3. **语义嵌入（sentence-transformers）**：计算文章与各类别描述的余弦相似度，取 Top-K 候选（阈值 0.45）
4. **GPT 最终确认**：将候选类别传给 GPT（gpt-4o-mini），批量确认分类结果；失败条目重试一次

**翻译：**
- 使用 GPT API 将英文标题和摘要翻译为中文
- 标题批量大小 12，摘要批量大小 3
- 摘要超过 800 字符时截断翻译

**输出：**
- `.xlsx`：每个分类对应一个 Sheet，同一篇文章可出现在多个 Sheet
- `.docx`：按分类分块排列，包含中英文标题、摘要、来源、日期、DOI

---

### 5. Word 导出 (`best_csv_to_word.py`)

- 使用本地 `Helsinki-NLP/opus-mt-en-zh` 离线翻译模型（Hugging Face）
- 生成格式规范的 `.docx` 文档，含中英对照摘要

## 技术栈

| 组件 | 技术 |
|------|------|
| RSS 解析 | `feedparser` |
| 网页抓取 | `playwright`（headless Chromium） |
| 学术元数据 API | Crossref、Semantic Scholar、OpenAlex、PubMed eUtils |
| 语义相似度 | `sentence-transformers` (all-MiniLM-L6-v2) |
| 翻译（离线） | `Helsinki-NLP/opus-mt-en-zh`（Hugging Face Transformers） |
| 翻译+分类（在线）| OpenAI GPT API（gpt-4o-mini） |
| 数据处理 | `pandas` |
| Word/Excel 生成 | `python-docx`、`openpyxl` |
| 自动化调度 | GitHub Actions（每日 UTC 定时触发） |

---

## 数据流说明

每天由 GitHub Actions 自动运行爬虫，结果 commit 到仓库（commit message 格式：`chore: daily RSS results (UTC YYYY-MM-DD)`）。周报在每周末手动或自动触发聚合与分类流程，生成最终的 `.xlsx` + `.docx` 文件供内部使用。
